# 振り返り
Local Storageの仕様に苦しめられました。おこづかい帳にするためには、一つのKeyに対して複数のデータを持たせる必要があったのですが、それができず。Object化することも試してみたのですが、そこからデータを取り出すのがかなりややこしくなって断念しました（ここまでで１日つぶしました。。。）。Keyとdataが一対一になるようなおこづかい帳の仕組みにするため、収入と支出をわけるようにしたところ、もろもろシンプルなものになりました。あまり、やりたいことありきで進めるのではなく、使うものの仕様に合わせて作るのも大事だと思いました。。
# 工夫点
同じKeyでデータがいれられないので、内容を入力するところでエラーを出すようにしたところなど、Local storageが仕様で対応できないところは、入力側で制御するようにした。項目を入力した数値の変更、削除をそれぞれのデータにつけるようにしたところ。
# 疑問点・苦労した点
最終的にできなかったこと：データを入力して入力ボタンを押した後に入力フィールドから文字を消したかった。データの並びがバラバラになってしまう。時系列にいれられなかった。POPUPの出現場所はこれでよかったのか？一括設定してるが、それぞれに設定して場所はそれぞれのところに出すほうがよかったか？
